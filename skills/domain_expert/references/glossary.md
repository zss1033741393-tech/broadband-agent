# 术语表

## 核心术语

| 术语 | 全称 | 说明 |
|------|------|------|
| **CEI** | Customer Experience Index | 用户体验指数，综合衡量网络体验质量 |
| **NCE** | Network Cloud Engine | 华为网络云化引擎，网络管理和编排平台 |
| **ONT** | Optical Network Terminal | 光网络终端，即光猫 |
| **QoS** | Quality of Service | 网络服务质量，用于流量优先级控制 |
| **RTT** | Round Trip Time | 往返时延，延迟的度量指标 |
| **OBS** | Open Broadcaster Software | 开源直播推流软件 |

## 方案术语

| 术语 | 说明 |
|------|------|
| **意图目标（IntentGoal）** | 用户需求的结构化表示，包含用户类型、场景、保障时段等 |
| **方案模板** | 预定义的配置参数框架，Agent 根据意图目标填充具体参数 |
| **约束校验** | 验证方案参数是否满足设备性能和策略冲突限制 |
| **配置转译** | 将语义化方案参数转换为设备可直接下发的配置格式（NL2JSON） |
| **闭环处置** | 发现问题后自动或半自动修复，形成感知→诊断→处置的完整闭环 |

## 用户类型

| 术语 | 说明 |
|------|------|
| **主播用户** | 进行直播推流的用户（抖音/快手/虎牙等平台），主要诉求为稳定上行带宽 |
| **游戏用户** | 以在线游戏为主要诉求的用户，对延迟和抖动高度敏感 |
| **VVIP用户** | 高价值大客户，默认高优先级保障，享受专属服务策略 |

## 套餐类型

| 术语 | 说明 |
|------|------|
| **普通套餐** | 标准宽带套餐，适合家庭场景，保守保障策略 |
| **直播套餐** | 面向直播场景的专属套餐，支持 STA 级保障和卖场/楼宇场景 |
| **专线套餐** | 高保障专线，上行带宽预留更高（≥90%），适合高价值直播用户 |

## 业务场景

| 术语 | 说明 |
|------|------|
| **家庭直播用户** | 在家中进行直播，主要痛点为 WiFi 覆盖弱导致上行卡顿 |
| **卖场走播场景** | 在商场、展厅等复杂环境移动直播，痛点为覆盖弱、强干扰、漫游断线 |
| **楼宇直播** | 在楼宇/商业综合体直播，痛点为 PON 接入层拥塞和云服务分发延迟 |

## 保障对象

| 术语 | 说明 |
|------|------|
| **家庭网络** | 以家庭整体网络为保障粒度，不区分具体设备 |
| **STA级** | 精确到特定终端设备（Station）的保障，需开启 per_user 感知模式 |
| **整网** | 面向整个部署网络的保障，适合卖场走播和楼宇场景 |

## 网络术语

| 术语 | 说明 |
|------|------|
| **上行带宽** | 从用户到网络的数据传输速率，直播推流主要消耗上行 |
| **下行带宽** | 从网络到用户的数据传输速率，视频播放主要消耗下行 |
| **WiFi 漫游优化** | 多个 AP 之间无缝切换，保持连接不中断，卖场走播场景必须开启 |
| **覆盖优化** | 调整 AP 发射功率和信道，改善覆盖范围，与漫游优化互斥 |
| **PON 拥塞** | 无源光网络接入层上行带宽共享争抢，导致推流卡顿，楼宇直播常见问题 |
| **云服务分发问题** | CDN 推流至云端后分发链路延迟或丢包，楼宇直播常见问题 |
| **APPflow** | 基于应用特征的流量精细化管理策略 |
| **节能策略** | 在业务低峰期降低设备功耗，通常在凌晨触发，需避开保障时段 |
