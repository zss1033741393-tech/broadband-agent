# 场景决策树

根据用户类型和套餐类型确定业务场景、典型问题及保障对象范围。

## 决策树

```
用户类型
├── 主播用户
│   ├── 普通套餐
│   │   └── 场景：家庭直播用户
│   │       典型问题：覆盖弱
│   │       保障对象：家庭网络 / STA级（指定设备）
│   │       保障时段：指定时段 或 全天
│   │       投诉记录：有 / 无
│   │
│   ├── 直播套餐
│   │   ├── 场景：卖场走播场景
│   │   │   典型问题：覆盖弱、强干扰、漫游问题
│   │   │   保障对象：STA级 / 整网
│   │   │   保障时段：指定时段 或 全天
│   │   │   投诉记录：有 / 无
│   │   │
│   │   └── 场景：楼宇直播
│   │       典型问题：PON拥塞、云服务分发问题
│   │       保障对象：STA级 / 整网
│   │       保障时段：指定时段 或 全天
│   │       投诉记录：有 / 无
│   │
│   └── 专线套餐（同直播套餐场景，上行带宽保障更高）
│       ├── 场景：卖场走播场景
│       │   保障对象：STA级 / 整网
│       └── 场景：楼宇直播
│           保障对象：STA级 / 整网
│
├── 游戏用户（套餐→场景决策树结构同主播用户）
│   ├── 普通套餐 → 家庭直播用户（关注低延迟/抖动）
│   ├── 直播套餐 → 卖场走播场景 / 楼宇直播
│   └── 专线套餐 → 卖场走播场景 / 楼宇直播
│
└── VVIP用户（套餐→场景决策树结构同主播用户，优先级默认 high）
    ├── 普通套餐 → 家庭直播用户
    ├── 直播套餐 → 卖场走播场景 / 楼宇直播
    └── 专线套餐 → 卖场走播场景 / 楼宇直播
```

## 场景说明

### 场景1：家庭直播用户（普通套餐）
- 用户在家中进行直播，主要痛点是 WiFi 覆盖不足导致上行卡顿
- 保障粒度：可选家庭整网，也可针对特定设备（STA级）
- 典型诉求：保障直播时段稳定的上行带宽

### 场景2：卖场走播场景（直播套餐 / 专线套餐）
- 用户在商场、展厅等复杂环境中移动直播
- 痛点：信号覆盖弱、多 AP 切换导致漫游中断、强同频干扰
- 保障粒度：STA 级（精确保障直播设备）或整网（全场覆盖）
- 专线套餐在此场景下上行带宽预留更高（>= 90% 上行保障）

### 场景3：楼宇直播（直播套餐 / 专线套餐）
- 用户在楼宇/商业综合体等场景直播
- 痛点：PON 接入层拥塞（共享上行带宽不足）、CDN 推流至云服务端分发延迟
- 保障粒度：STA 级 或 整网
- 需关注 PON 拥塞监控和云端分发链路质量

## 推断提示

| 用户描述关键词 | 推断场景 |
|--------------|---------|
| 家里直播 / 家庭网络 / 覆盖差 | 家庭直播用户（普通套餐） |
| 卖场 / 走播 / 商场 / 展厅 / 漫游 | 卖场走播场景（直播套餐或专线套餐） |
| 楼宇 / 商业体 / PON / 推流卡 | 楼宇直播（直播套餐或专线套餐） |
| 专线 / 高保障 / 高优先 | 专线套餐 |
